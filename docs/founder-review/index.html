Woobly · Meeting Room Management

Founder Review — Product Wireframes

Low-fidelity, grayscale wireframes of the core operator and guest screens. These communicate layout, hierarchy, workflow, and information architecture — the thinking that precedes high-fidelity Figma, not finished UI.

Grayscale · structure-only · open any file in a browser

Woobly is the smart-hospitality platform. Quorum is the client venue (the guest-facing brand shown on the in-room LUXEGENIE tablet). Management books meeting rooms through a guided calendar-first flow and runs the day from a room-first live board. At the slot time a room shows Reserved; the guest taps Start Meeting on LUXEGENIE → In-Use. Guests request Wi-Fi, assistance, F&B, extensions and the bill; staff service each from the dashboard. Meetings never end automatically — a person closes them, then the room returns to Available and the next booking surfaces.

The room lifecycle drives everything

Available→Reserved→In-Use→ Ending Soon→Billing→Available ·  Under Maintenance = interrupt

How to read these wireframes

{ State }A visible state token. Status is shown by text, never colour alone (colour arrives in Figma).
[ Button ]A control. A dark-filled button is the single primary action for that surface/state.
⌕ SearchSearch input, always top-left of a list surface.
( Tab )Segmented filter tab; the filled one is active.
⋮Overflow menu holding secondary / management-authority actions.
Drawer / ModalA right-side drawer or centered modal overlays the surface behind it — it never navigates away.

What each screen is designed to prove

  • One coherent system — identical shell, sidebar, spacing, type hierarchy, cards, drawers and buttons across every screen.
  • Reduce staff effort — one obvious action per state; attention rises to the top; the engine matches rooms so staff never scan a list; details auto-fill.
  • Room-first & calendar-first — the live board is the landing surface; booking is a guided constraint→availability sequence.
  • Every state is drawn — default, loading, empty, error, escalation, and overlays are represented, not implied.
